## P1 項目（STM32 GATTC branch `6c9b2a1`）

- **case 0x80 (disconnect)** 和 **case 0x83 (scan timeout)** 都呼叫 `start_stop_scan_pedal(PEDAL_CMD_START_SCAN)` (0x01)
  - Design 明示要 `PEDAL_CMD_RECONNECT` (0x10)
  - START_SCAN 會讓 ESP32 **clear NVS + use_mac_filter=false** → 一斷線就清掉配對
- 兩處還設 `set_bt_long_pressed(1)` — 模擬手動配對，進一步破壞 auto-reconnect 語意
- **沒 backoff** — Design 要求 scan_timeout_count: 1-3 立刻 / 4-10 delay 30s / >10 delay 120s，全缺
- **0x83 handler 缺 `bt_long_pressed` guard**（AI 建議：防止打斷手動配對）
- GATTC branch base 很舊，rebase 到 develop 時會帶進/拿掉大量 factory preset inc 檔案

## P2 項目（觀察，非 bug）
