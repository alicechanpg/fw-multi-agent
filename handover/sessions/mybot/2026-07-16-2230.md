## 本次完成

### FWP-814 — Spark LIVE audio 偵測不到（root cause + fix）
- **Root cause**：audio anchor 假設所有 audio chip 都是 PG VID。`findSparkPid()` 只認 `vid == kUsbVid (0x295d)`，且 `getSparkLiveConfig().audioPid = "0221"`。
  **Spark LIVE 的 audio 其實是 C-MEDIA `0d8c:0033`** → 永遠比對不到 → fallback 回 Reactor 預設（Reactor 圖 + Reactor 文案 + Reactor DFU PID）。
- **`0221` 的來源**：我自己的 commit `ffdd3dc` 從 Spark 2 的 `0231` 推測出來的，repo 內 grep 不到任何 spec 依據。
- **Fix `938239e`**（已 push）：audioPid 改成 `"vid:pid"`（純 `"pid"` 代表 PG VID）→ Spark 2 / EDGE 不受影響。
- **Review 第二輪抓到的 regression**：`0d8c:0033` 是**通用 C-MEDIA 晶片**，一般 USB 音效卡也會回報；而且原本 Spark 比 Reactor 先偵測 → 一顆便宜 USB DAC 會被當成 LIVE 並蓋過真正接著的 Reactor。已改為 **PG VID 優先** + lookup 以 ID 為外層迴圈 + 先驗 Reactor BT Audio。
